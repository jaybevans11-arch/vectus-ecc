🏗️ What This Is

A custom-built estimating management dashboard for the Senior Estimator role. Built to solve one real problem: managing 6–15 simultaneous RFQs without dropping the ball on deadlines, follow-ups, or client commitments.

Before this tool, tracking was scattered across a notebook, iPad notes, and Smartsheet — no single source of truth, no way to see what needed attention today. This consolidates everything: project tracking, checklist discipline, schedule gates, scorecard measurement, and personal growth — in one place, synced to Google Sheets, accessible from any device.

Core principle: Every project always has a clear Next Action. Work Today's Focus top-to-bottom each morning and you are never reactive — you are always ahead.
⚙️ How It's Built
Frontend
A single HTML file — no frameworks, no installation. Opens in any browser on any device.
Hosting
GitHub Pages — permanent URL, zero cost, accessible from any device at any time.
Database
Google Sheet "ESTIMATING" — one row per project, human-readable, exportable. This is the single source of truth for all project data.
Sync Layer
Google Apps Script bridges the dashboard and the Sheet. Every save syncs in real time. On load, the Sheet feeds the dashboard.
Local Storage
Scorecard data and Growth streaks are stored in the browser's local storage. Don't clear your browser cache — this data does not sync to Sheets.
Cost
$0. GitHub Pages, Google Sheets, and Apps Script are all free. No subscriptions, no vendor lock-in.
Data flow: Dashboard → Apps Script → Google Sheets. On load: Sheets → Script → Dashboard. Scorecard and Growth data live in the browser only.
📅 Daily Workflow
Morning — 5 min
Open Today's Focus. Check your growth nudge — pick one if not set. Work the project list top-to-bottom: overdue first, due today second, high priority third.
New RFQ arrives
Click + New RFQ. Enter project code, contact, value, and submission date. Set Key Dates (freeze gate, submission-ready gate) and click 📅 Outlook on each. Set the first Next Action. Save. The project is live and synced instantly.
During the day
Open any project. Tick checklist items. Update Notes. Log risks, assumptions, and blockers as they arise — don't hold them in your head. Before closing: always update Next Action. This is the one habit everything else depends on.
End of day — 5 min
Scan Pipeline. Every active project needs a current Next Action. Any ⚠ AT RISK flag means either schedule the time or raise the capacity issue now — not later.
Friday — 5 min
Go to Scorecard. Click ⚡ Log This Week. The system auto-fills # estimates, stage gate %, peer review %, freeze %, and sub-ready % from your submitted projects. The only field you fill manually: Defects. Add notes and save.
Monday morning — 10 min
Open This Week. The schedule distributes project work across Mon–Fri by priority and deadline. Copy blocks into Outlook. Send last week's scorecard row to leadership.
The critical habit: Next Action must be current before closing any project. If it's blank or stale, the project disappears from your priority radar. Everything else in the system depends on this one field.
📋 Tiered Checklist System

The checklist automatically adjusts based on project value the moment you enter it. Each tier gets the right amount of process — no more, no less.

⚪ Micro — under $5K (~2h total)
Client Setup → Day 0 → Day 1 → Site Visit (if needed) → Estimating (takeoff + materials) → Submission. Leanest process — no sub quotes, no scope matrix, no sign-off, no submission prep checklist.
⚫ Small — $5K–$10K (~3h total)
Same lean flow as Micro with slightly more time budgeted for scope and pricing. No sign-off required. Takeoff + materials only.
🔵 Mid — $10K–$20K (~4.5h total)
Full estimating process: scope matrix, takeoff, RFQs, pricing basis, narrative, checkpoint. Full Submission & Presentation checklist. No internal sign-off required at this tier.
🟡 Standard — $20K–$50K (~7h total)
Everything in Mid + internal Review & sign-off required. The 24-hour freeze gate applies. Peer review scheduled before freeze.
🔴 Large — $50K+ (~11h total)
Everything in Standard + Closeout & Continuity (archive, 1-page summary, handoff, defect log). In-person presentation logistics included.
⭐ Specialty — any value
Inherits the Large checklist and adds Secondary Review and Final Sign-off steps. Use for high-complexity or high-risk jobs regardless of dollar value.
A tier badge appears at the top of every checklist so you always know which process is active. Hours are weighted — a 5-minute gate task moves the progress bar far less than a 90-minute site visit. Progress % is real, not just item count.
📅 Key Dates & Schedule Gates

Every project has a Key Dates section in its modal. This is where schedule gates live — not in the checklist. Set them once at intake. Click 📅 Outlook on each gate to block it in your calendar immediately.

📬 Submission / Presentation Date
The hard deadline — the date the quote is due to the client. Everything else is anchored to this date.
🔒 Internal Freeze Gate (48hrs before submission)
The estimate is locked. No more changes. This gate is what the Freeze % in the Scorecard measures — did you hit it?
✅ Submission-Ready Gate (24hrs before submission)
PDF verified, copies counted, cover email drafted. Everything is ready to go with 24 hours to spare.
👤 Peer Review Scheduled
Large and specialty projects only. The reviewer is named and the time is blocked before the freeze gate.
Other Gates
Site Visit, RFQ Decision Deadline, Follow-Up Call, and Client's Completion Date are also available — use what applies to each job.
📊 Scorecard — Weekly Measurement

Every Friday, click ⚡ Log This Week in the Scorecard view. The system auto-calculates all metrics from your submitted projects that week. You fill in one field manually: Defects.

Stage Gate %
Average checklist completion across all submitted projects this week. Target: ≥95%.
Peer Review %
% of large/specialty projects with internal review completed (c14 ticked). Small jobs count as passed automatically. Target: 100%.
Freeze %
% of projects where the freeze date was set at or before the submission deadline. Target: ≥90%.
Sub-Ready %
% of projects where the submission-ready date was set before the submission deadline. Target: ≥90%.
Defects ✎
Manual entry only — errors found after freeze. Use the + Log Defect button in the final checklist phase to increment this instantly without switching views. Target: 0.
30/60/90 Milestones
Four milestones tracking adoption of the system over the first 90 days. Update status manually each week as you progress.
Note: Scorecard and milestone data are saved in your browser's local storage. Don't clear your browser cache on the device you use for this dashboard.

Marking done

What to focus on
Your R3 report flags: decision-making without analysis-paralysis, asserting yourself in meetings, sharing reasoning out loud, and communicating under stress instead of going quiet. These are the areas where one daily rep compounds fastest.
🔒 Security & Privacy
  • Password protected on every load — no data is visible without the correct password.
  • No client company names are stored. Projects use internal code names only (e.g. PROJ-2025-047).
  • Contact name and phone/email are stored — equivalent to what lives in Outlook or a notebook.
  • The Google Sheet is private to the owning Google account and is not publicly shared.
  • The dashboard URL is technically public but the password prevents any data access.
Tip: For sensitive projects, use abstract codes (e.g. NW-COMM-04) rather than descriptive names. The system is designed to support this.

🔗 Outlook Calendar Connection — How It Works & Security

When you click any 📅 Outlook button on Key Dates or in Tasks, it opens a pre-filled event in Outlook Web. Nothing is sent automatically — you review the event and click Save to confirm it in your calendar.

How it works: The dashboard builds a standard Microsoft Outlook deep-link URL with the event details (title, date, time, description) encoded into the web address. Clicking it opens Outlook in a new tab on your screen. Your calendar credentials are never touched by this tool.
📅 Key Date Bookings (Projects)
Submission date, Freeze Gate, Sub-Ready Gate, Peer Review, Site Visit, and RFQ Deadline — each Outlook button pre-fills the event with the project name, date, and a plain-English description of what that gate means.
✅ Task Bookings
Set a due date on any task and a "Book in Outlook" button appears in the edit form. It creates a 30-minute reminder block at 9:00 AM on that date, with the task name, category, and notes pre-filled.

What has access to what:

  • ✅ The event details you see on screen are passed via the URL to Outlook
  • ✅ Your Outlook calendar — once you're logged in at office.com, it saves to your calendar
  • ❌ Outlook cannot see your Google Sheets data, project list, or scorecard
  • ❌ The dashboard does not interact with your Outlook credentials in any way
  • ❌ No data is sent to any server when you click an Outlook button — it's a one-way link
Security summary: This dashboard uses no OAuth tokens, no third-party authentication, and no stored calendar credentials. The Outlook buttons are plain hyperlinks. Your data lives in your Google Sheet (which you control) and this HTML file. Nothing is shared with external services except when you explicitly click a button to open Outlook in your browser.
