How This Works
Built for EDL Building Solutions — Estimating Department
🏗️ What This Is

This is a custom-built estimating management dashboard designed specifically for the Senior Estimator role at EDL Building Solutions. It was built to solve a real problem: managing 6–15 simultaneous RFQs across multiple devices without dropping the ball on deadlines, follow-ups, or client commitments.

Before this tool, project tracking was scattered across a notebook, iPad notes, and Smartsheet — with no single source of truth and no way to see at a glance what needed attention today. This dashboard consolidates everything into one place, accessible from any device.

Core principle: Every project always has a clear "Next Action." If you work top-to-bottom on Today's Focus each morning, you are never reactive — you are always ahead.
⚙️ How It's Built
Frontend
A single HTML file with embedded CSS and JavaScript. No frameworks, no installation required. Opens in any browser — Chrome, Safari, Edge.
Hosting
Hosted on GitHub Pages at no cost. The URL is permanent and accessible from any device — work desktop, home computer, or iPad.
Database
All project data is stored in a Google Sheet called "EDL ESTIMATING." Every save, edit, and delete syncs in real time. The sheet is the single source of truth for all records.
Sync Layer
A Google Apps Script web app bridges the dashboard and the Google Sheet. When you save a project, the data is sent to the script, which writes it to the sheet — and vice versa on load.
Cost
$0. GitHub Pages is free. Google Sheets and Apps Script are free. No subscriptions, no server costs, no vendor lock-in.
Maintenance
The dashboard file can be updated at any time by uploading a new version to GitHub. No IT involvement required. Data in Google Sheets is unaffected by updates.
Data flow: Dashboard (browser) → Google Apps Script → Google Sheets. On load, the flow reverses: Sheets → Script → Dashboard.
🔒 Security & Privacy

The dashboard is protected by a password screen on every load. Without the correct password, no data is visible.

  • No client company names are stored. Projects use internal code names only (e.g. EDL-2025-047).
  • Contact name and phone/email are stored — equivalent to what lives in Outlook or a notebook.
  • The Google Sheet is private to the owning Google account and is not publicly shared.
  • The Apps Script only accepts structured requests from the dashboard — it cannot be browsed externally.
  • The dashboard URL is technically public but the password prevents any data access.
Tip: For sensitive projects, use abstract codes (e.g. "NW-COMM-04") rather than descriptive names. The system is designed to support this.
📅 Daily Workflow
Morning — 5 min
Open Today's Focus. Work top-to-bottom. Overdue items appear first in red, due-today in blue. Each project shows weighted progress, hours remaining, and current next action.
New RFQ arrives
Click + New RFQ in the sidebar. Enter project code, contact details, estimated value, and quote deadline. Set the first Next Action. Save. The project is live and synced to Google Sheets instantly.
During the day
Click any project to open it. Drop notes into the Notes field. Tick checklist items as they're completed. Before closing, always update the Next Action field. This is the habit that makes the whole system work.
End of day — 5 min
Scan the Pipeline view. Every active project should have a current next action. Any project marked ⚠ AT RISK needs attention — either schedule the time or flag the capacity issue.
Monday morning — 10 min
Open This Week. The system distributes project work across Mon–Fri based on priority, deadlines, and estimated hours. Copy blocks into Outlook to time-block your estimating week.
The critical habit: The Next Action field must always be current before closing a project. This single field drives Today's Focus. If it's blank or stale, the project disappears from your priority radar.
📋 Estimating Process & Checklist

Every project includes a process checklist organized into phases. Each item carries a time weight used to calculate true weighted progress — ticking "Acknowledge receipt" (5 min) moves the needle far less than completing a site visit (90 min).

Day 0 — Same Day
Acknowledge receipt, log in EDL Pipeline, create share drive folder. Light admin (~10 min total) but critical for starting correctly.
Day 1 — 24hrs
Scope investigation, first client call, determine site visit need. Time scales with project value — 30 min for a $5K RFQ, 2+ hours for a $200K project.
Site Visit
Book, execute, consolidate notes and photos. One of the heaviest phases — typically 1.5–2 hours including travel, time on site, and documentation.
Estimating
Quantity takeoff, material and labour costs, subcontractor quotes. The core of the work. Scales with project size — 1 hour for small jobs, 8–10 hours for large commercial projects.
Review
Internal review and sign-off. Required for projects over $50K. Specialty projects receive two additional steps — Secondary Review and Final Sign-off.
Submission
Quote submission and follow-up scheduling. Large projects include in-person presentation prep. Follow-up is a separate tracked step.
Client Setup
Jonas setup, new client forms, credit application. Applies to first-time clients only — skipped for repeat business.
Quality Control
QC checklist review and documentation filing. Final gate before a project is marked complete.
📊 Progress Tracking & Capacity
Under $10K
~1–2 hours total estimated effort. Lean process — no internal review, lighter scope investigation, faster submission.
$10K–$50K
~2–4 hours total. Standard process without mandatory internal review. Most routine commercial estimates fall here.
$50K+
~6–14 hours total. Full process including internal review and in-person presentation. Requires the most calendar planning.
⭐ Specialty
Any project flagged Specialty receives two additional Review steps (+1.5 hrs) regardless of value, reflecting higher risk and complexity.

AT RISK ⚠: A project is flagged when remaining weighted hours exceed the available productive time before the deadline (assuming 6 hrs/day). This is an early warning designed to surface capacity problems before they become missed deadlines.

Capacity bars:

  • Overall Load — Active projects vs. max 15. Primary signal for workload conversations with management.
  • Active Estimates — Projects in any active stage vs. 15. High numbers mean high cognitive load even if deadlines are spread out.
  • Site Visits Pending — Projects awaiting site visit vs. max 4. Tracked separately because site visits require significant calendar blocking.
🔧 Maintaining & Updating
Updating the dashboard
Edit the HTML file and re-upload to GitHub. Changes go live within 60 seconds. No IT involvement required. Data in Google Sheets is never affected by a dashboard update.
Accessing data directly
The Google Sheet "EDL ESTIMATING" can be opened at any time to view, export, or back up all project records. One row per project, human-readable, exportable to Excel.
Adding team members
The dashboard uses a single shared password. Additional estimators can be given access by sharing the URL and password. The Google Sheet can be shared with management for read-only visibility.
Changing the password
Update the password in the dashboard file and re-upload to GitHub. The old password stops working immediately.
If sync stops working
The most common cause is the Apps Script authorization expiring. Re-open the script in Google Drive, re-deploy as a web app, and update the script URL in the dashboard file. Takes approximately 5 minutes.
Backup: All data lives in Google Sheets, which is automatically backed up by Google with full version history. No project record can be lost due to a browser crash, device failure, or dashboard update.
